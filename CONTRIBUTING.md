# contributing to safe-solana-builder

Thanks for helping improve this project. This repo is mainly documentation (the Cursor skill in `SKILL.md`, reference guides under `references/`, and example programs under `examples/`) plus a packaged desktop build linked from the readme.

## before you open a pull request

- **Fork** the repository and create a **branch** for your change (avoid committing directly to `main` on your fork if you plan to send PRs).
- **Keep changes focused** — one logical improvement per PR is easier to review than a mixed bag of unrelated edits.
- **Match existing tone** — security-focused, clear wording; follow the structure used in `SKILL.md` and the reference markdown files.

## what makes a good contribution

- Fixes to **incorrect or misleading instructions** (especially install/download paths and security notes).
- **Alignment** between `SKILL.md` and the `examples/` artifacts (e.g. security checklists that follow the same sections as the skill template).
- **Reference updates** when Solana, Anchor, or Metaplex best practices change — cite sources in the PR description when possible.
- **Example programs** that compile and include a `security-checklist.md` consistent with the skill.

## commit messages

Write short, descriptive messages in plain language. No need for a special prefix unless you prefer one.

## review process

Maintainers may ask for small follow-up edits. If something is unclear, opening an issue on the upstream repository before large work can save time.

## code of conduct

Be respectful and assume good intent. Technical disagreement is normal; keep discussion about the work, not the person.
